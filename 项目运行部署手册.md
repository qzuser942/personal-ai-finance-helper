## 5. Qdrant向量数据库部署

> **说明**：Qdrant用于存储用户消费行为向量，实现AI个性化分析记忆。若Qdrant不可用，后端不会崩溃，AI分析将跳过个性化记忆功能。

### 5.1 【必选步骤】Docker一键启动Qdrant

```bash
docker run -d \
  --name qdrant \
  --restart unless-stopped \
  -p 6333:6333 \
  -p 6334:6334 \
  -v qdrant_storage:/qdrant/storage \
  qdrant/qdrant:latest
```

**参数说明**：
| 参数 | 含义 |
|------|------|
| `--name qdrant` | 容器名称 |
| `--restart unless-stopped` | Docker重启时自动启动容器 |
| `-p 6333:6333` | REST API端口映射（健康检查用） |
| `-p 6334:6334` | **gRPC端口映射（后端使用）** |
| `-v qdrant_storage:/qdrant/storage` | 数据持久化到Docker Volume |

**验证命令**：
```bash
# 方法一：检查容器运行状态
docker ps --filter name=qdrant

# 方法二：检查健康端点
curl http://localhost:6333/health

# 方法三：查看容器日志
docker logs qdrant
```

**成功标准**：
- `docker ps` 输出中 `STATUS` 列为 `Up X seconds`
- `curl http://localhost:6333/health` 返回 `{"title":"healthz","version":"..."}`

---

### 5.2 【可选步骤】Qdrant常用管理命令

```bash
# 停止Qdrant
docker stop qdrant

# 启动已存在的Qdrant容器
docker start qdrant

# 重启Qdrant
docker restart qdrant

# 查看Qdrant日志
docker logs -f qdrant

# 删除Qdrant容器及数据（⚠️ 不可恢复）
docker stop qdrant && docker rm qdrant
docker volume rm qdrant_storage
```

## 6. 大模型API密钥配置

> **说明**：项目使用**DeepSeek官方云端API**（`https://api.deepseek.com/v1`）进行大模型推理，使用**阿里云百炼DashScope**（`text-embedding-v2`）进行文本嵌入向量化。

### 6.1 【必选步骤】获取DeepSeek API Key

```bash
# 1. 访问 DeepSeek 开放平台
# https://platform.deepseek.com/

# 2. 注册账号并登录

# 3. 进入「API Keys」→「创建 API Key」

# 4. 复制生成的密钥（格式：sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx）

# 5. ⚠️ DeepSeek API为付费服务，需先充值（最低充值10元）
#    计费参考：deepseek-chat 约 ¥1/百万tokens (输入) + ¥2/百万tokens (输出)
```

🔴 **生产环境必改项**：必须使用自己的API Key，不可使用占位值。

---

### 6.2 【必选步骤】获取阿里云百炼DashScope API Key

```bash
# 1. 访问阿里云百炼控制台
# https://bailian.console.aliyun.com/

# 2. 开通「模型服务灵积」→「文本嵌入」

# 3. 在模型广场确认 text-embedding-v2 模型已授权

# 4. 进入「API-KEY 管理」→ 创建API Key

# 5. 复制生成的密钥（格式：sk-xxxxxxxxxxxxxxxx）
```

🔴 **生产环境必改项**：必须使用自己的API Key，不可使用占位值。

---

### 6.3 【必选步骤】配置API Key

**推荐方式：使用环境变量**（密钥不写入配置文件，安全性更高）

**Windows PowerShell**：
```powershell
# 临时生效（当前终端窗口）
$env:DEEPSEEK_API_KEY="sk-your-actual-deepseek-key"
$env:DASHSCOPE_API_KEY="sk-your-actual-dashscope-key"

# 永久生效（系统环境变量）
[System.Environment]::SetEnvironmentVariable('DEEPSEEK_API_KEY', 'sk-your-actual-deepseek-key', 'User')
[System.Environment]::SetEnvironmentVariable('DASHSCOPE_API_KEY', 'sk-your-actual-dashscope-key', 'User')
```

**macOS / Linux Terminal**：
```bash
# 临时生效
export DEEPSEEK_API_KEY="sk-your-actual-deepseek-key"
export DASHSCOPE_API_KEY="sk-your-actual-dashscope-key"

# 永久生效（追加到 ~/.bashrc 或 ~/.zshrc）
echo 'export DEEPSEEK_API_KEY="sk-your-actual-deepseek-key"' >> ~/.bashrc
echo 'export DASHSCOPE_API_KEY="sk-your-actual-dashscope-key"' >> ~/.bashrc
source ~/.bashrc
```

**备选方式：直接修改 `application.yml`**（仅开发环境推荐）：

```yaml
# 编辑文件: personal-finance-server/src/main/resources/application.yml
ai:
  deepseek:
    api-key: sk-your-actual-deepseek-key   # 🔴 填入真实Key
  alicloud:
    dashscope-api-key: sk-your-actual-dashscope-key  # 🔴 填入真实Key
```

**验证命令**：
```bash
# 确认环境变量已设置
# Windows PowerShell
echo $env:DEEPSEEK_API_KEY
echo $env:DASHSCOPE_API_KEY

# Linux/Mac
echo $DEEPSEEK_API_KEY
echo $DASHSCOPE_API_KEY
```
**成功标准**：输出以 `sk-` 开头的密钥字符串。

### 11.2 Linux/macOS启动停止脚本

**文件**：`start.sh`

```bash
#!/bin/bash
# 个人智能理财系统 - 启动脚本

APP_NAME="personal-finance-server"
APP_JAR="personal-finance-server/target/personal-finance-server-1.0.0.jar"
APP_LOG="logs/app.log"
APP_PID_FILE="app.pid"

echo "========================================"
echo "  个人智能理财系统 - 启动"
echo "========================================"

# 切换到脚本所在目录
cd "$(dirname "$0")"

# 检查是否已在运行
if [ -f "$APP_PID_FILE" ]; then
    PID=$(cat "$APP_PID_FILE")
    if ps -p "$PID" > /dev/null 2>&1; then
        echo "[警告] 服务已在运行中 (PID: $PID)"
        exit 1
    fi
fi

# 检查Java
if ! command -v java &> /dev/null; then
    echo "[错误] 未找到Java环境"
    exit 1
fi

# 检查jar包
if [ ! -f "$APP_JAR" ]; then
    echo "[信息] jar包不存在，开始编译..."
    cd personal-finance-server
    ./mvnw clean package -DskipTests
    cd ..
    if [ $? -ne 0 ]; then
        echo "[错误] 编译失败"
        exit 1
    fi
fi

# 创建日志目录
mkdir -p logs

# 启动服务
echo "[信息] 启动服务..."
nohup java -jar "$APP_JAR" > "$APP_LOG" 2>&1 &
echo $! > "$APP_PID_FILE"

echo "[成功] 服务已启动 (PID: $(cat $APP_PID_FILE))"
echo "[信息] 日志文件: $APP_LOG"
echo "[信息] Knife4j文档: http://localhost:8080/doc.html"

# 等待启动完成
sleep 5
if ps -p "$(cat $APP_PID_FILE)" > /dev/null 2>&1; then
    echo "[成功] 服务启动验证通过"
    tail -5 "$APP_LOG"
else
    echo "[错误] 服务启动失败，请查看日志: $APP_LOG"
    exit 1
fi
```

#!/bin/bash
# 个人智能理财系统 - 停止脚本

APP_PID_FILE="app.pid"

cd "$(dirname "$0")"

if [ -f "$APP_PID_FILE" ]; then
    PID=$(cat "$APP_PID_FILE")
    if ps -p "$PID" > /dev/null 2>&1; then
        echo "正在停止服务 (PID: $PID)..."
        kill "$PID"
        sleep 3
        if ps -p "$PID" > /dev/null 2>&1; then
            echo "强制终止..."
            kill -9 "$PID"
        fi
        echo "服务已停止"
    else
        echo "服务未在运行"
    fi
    rm -f "$APP_PID_FILE"
else
    echo "未找到PID文件"
fi

## 13. 常见问题排查

### 13.1 后端启动失败

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| `Connection refused: connect` | MySQL未启动或端口错误 | `mysql -u root -p` 测试连接；检查`application.yml`中端口 |
| `Access denied for user 'root'@'localhost'` | 数据库密码错误 | 检查`application.yml`中`spring.datasource.password` |
| `Unknown database 'finance_db'` | 数据库未创建 | 执行第4章数据库初始化脚本 |
| `Port 8080 was already in use` | 端口被占用 | `netstat -ano \| findstr 8080` 查占用进程，或换端口 |
| `Java version mismatch` | JDK版本错误 | 必须JDK 17+；`java -version` 确认 |
| `Cannot find symbol ... QdrantClient` | Qdrant依赖未加载 | `mvn clean compile` 重新下载依赖 |

### 13.2 MySQL连接问题

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 后端反复报`CommunicationsException` | 连接超时 | 检查`spring.datasource.druid.max-wait` 和 MySQL `wait_timeout` |
| `Table 'finance_db.bill' doesn't exist` | 建表脚本未完整执行 | 重新执行第4章全部建表SQL |
| `Cannot add foreign key constraint` | 外键依赖表未先创建 | 注意建表顺序：先`category`再`bill`添加外键 |

### 13.3 Qdrant向量库问题

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 启动日志无Qdrant初始化信息 | Docker未启动或Qdrant容器未运行 | `docker start qdrant` |
| `UNAVAILABLE: io exception` | Qdrant gRPC端口6334不可达 | `docker ps \| grep qdrant` 检查端口映射 |
| AI分析正常但无个性化记忆 | Qdrant连接失败 | 后端自动降级，不影响核心功能 |
| `Collection user_consumption_vectors already exists` | 正常日志，非错误 | 后端会自动检查集合是否存在，无需处理 |

### 13.4 AI模块问题

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| AI分析返回"AIServiceException" | DeepSeek API Key未配置或无效 | 确认环境变量`DEEPSEEK_API_KEY`已设置且以`sk-`开头 |
| AI分析返回"401 Unauthorized" | API Key格式错误 | 到DeepSeek平台重新生成Key |
| AI分析返回"429 Too Many Requests" | API调用频率超限 | 等待1分钟后重试；联系DeepSeek提升配额 |
| 向量嵌入返回空 | 阿里云百炼API Key未配置或无效 | 确认环境变量`DASHSCOPE_API_KEY`已设置 |
| 分析结果为空或格式不对 | Prompt模板问题 | 管理员后台检查`ai_config`表中Prompt模板 |

### 13.5 JWT鉴权问题

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 接口返回401或code=10004 | Token过期或无效 | 重新登录获取新Token |
| Token验证失败 | JWT密钥不匹配（多实例部署） | 确保所有实例使用相同的`jwt.secret` |
| 管理员接口报403 | 使用了普通用户Token | 普通用户和管理员Token不能混用 |
| 用户接口报403 | 未携带Token | 请求头添加`Authorization: Bearer <TOKEN>` |

### 13.6 管理后台问题

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 页面白屏 | 前端环境变量配置错误 | 检查`.env.development`中API地址 |
| 接口报CORS错误 | 后端跨域配置问题 | 检查`WebMvcConfig.java`中的CORS配置 |
| 登录成功但跳回登录页 | Token未正确存储 | 检查浏览器localStorage中`admin_token` |
| `Module not found` 编译报错 | npm依赖未安装 | 重新执行`npm install` |

### 13.7 鸿蒙端问题

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 编译报`SDK版本不匹配` | DevEco Studio SDK版本不符 | 确认`build-profile.json5`中`compileSdkVersion`匹配已安装SDK |
| 手机无法连接后端 | 手机和电脑不在同一局域网 | 确认WiFi一致；检查防火墙 |
| 请求超时 | 后端地址填错或网络不通 | 先用手机浏览器打开`http://电脑IP:8080/doc.html`测试 |
| 离线记账后无法同步 | 网络未恢复或UUID冲突 | 检查WiFi连接；查看后端日志同步接口报错 |

### 13.8 跨域问题

| 现象 | 原因 | 解决方案 |
|------|------|----------|
| 浏览器Console报CORS | 源不在允许列表中 | 后端已配置`allowedOriginPatterns("*")`，如仍报错检查拦截器是否异常 |
| OPTIONS预检请求失败 | 跨域预检未通过 | 检查后端`WebMvcConfig`中是否允许OPTIONS方法 |
| Cookie/Session丢失 | `Access-Control-Allow-Credentials` 未配置 | 后端已配置`allowCredentials(true)` |
