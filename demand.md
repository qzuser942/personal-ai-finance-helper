

# 精简可直接投喂AI · 理财App前后端+管理员后台 PMF完整需求清单
## 整体架构
鸿蒙ArkTS手机客户端 + SpringBoot电脑局域网后端 + Web管理员后台 + MySQL + LangChain4j本地大模型 + Qdrant向量库
数据全部持久化存储在电脑MySQL；手机无本地RDB数据库，离线账单缓存于手机Preferences，联网后一键同步至电脑服务端；AI分析、统计计算、文件生成、后台运营管理全部交由电脑后端实现，手机仅做用户端UI交互。

## 一、鸿蒙ArkTS手机客户端完整功能
### 1. 首页记账录入页
1. 收支录入：区分收入/支出，填写金额、选择消费分类、文字备注，可选拍摄消费小票上传至电脑服务端
2. 快捷金额按钮，减少手动数字输入
3. 网络正常时直接提交账单至后端；无网络则缓存至本地离线队列，联网自动批量同步
4. 草稿保存、取消录入功能

### 2. 账单流水页面
1. 下拉刷新从电脑后端分页拉取全部账单，按消费时间倒序展示
2. 单条账单操作：查看详情、编辑内容、删除记录，修改实时同步电脑MySQL
3. 多维度筛选：指定月份、收支类型、消费分类、关键词检索账单
4. 一键请求后端生成账单Excel文件，下载保存至手机本地

### 3. 月度统计图表页（鸿蒙ECharts）
1. 展示后端聚合计算后的月度总收入、总支出、月度结余
2. 支出分类占比饼图、每日收支趋势柱状图，数据全部由后端计算返回
3. 月份切换，拉取对应历史月度财务统计数据

### 4. 预算与存钱目标页面
1. 从后端拉取当月总预算、各分类子预算，渲染可视化进度条
2. 消费金额接近/超出对应预算时，APP弹窗预警提示
3. 新建存钱目标、追加存款金额，数据实时同步电脑服务端
4. 存钱目标完成标记、归档查看历史目标

### 5. AI智能理财分析页（调用电脑本地大模型）
1. 一键请求后端，上传当月全部账单数据集
2. 后端使用DeepSeek大模型分析，返回冗余消费项、不良消费习惯、个性化省钱方案、月度财务复盘文案，前端展示排版
3. 智能分类推荐：输入消费备注，AI自动匹配对应收支分类

### 6. 设置与关于页面
1. 浅色/深色两套系统主题一键切换；应用密码锁保护账单隐私
2. 手动触发离线缓存账单同步、本地账单文件备份
3. 关于页面固定信息：姓名：胡宪棋，班级：软件2413，学号：202421332084，软件功能介绍

### 客户端硬性开发规范
1. 技术栈：ArkTS声明式UI、组件化开发、页面路由统一管理
2. 本地存储仅使用Preferences：缓存登录Token、离线待同步账单、主题配置、密码锁配置
3. 网络配置：开启INTERNET权限，统一请求地址 `http://电脑局域网IP:8080/接口路径`
4. 无本地RDB数据库，所有正式业务数据仅存储在电脑MySQL，本地仅临时缓存离线数据

## 二、SpringBoot电脑后端完整业务模块
### 1. 用户账户模块（普通用户）
1. 用户注册、账号密码登录接口，使用JWT签发身份令牌做鉴权
2. 用户密码BCrypt加密存储，用户信息增删改查
3. 数据隔离：所有账单、预算、目标数据绑定用户ID，不同用户数据互不干扰

### 2. 管理员账号模块（Web后台专用）
1. 管理员独立登录接口，独立JWT令牌，区分普通用户与管理员权限
2. 后台权限控制：超级管理员、运营管理员两级角色
3. 管理员账号CRUD、密码重置、权限分配接口

### 3. 账单核心业务模块（MySQL持久化）
1. 账单完整CRUD接口：新增、修改、删除、分页查询、多条件筛选
2. 消费分类管理：系统内置基础收支分类，支持用户自定义分类并持久化
3. 预算管理接口：按月创建、修改总预算、各分类子预算
4. 存钱目标CRUD接口，记录目标金额、已存金额、完成状态
5. 月度数据聚合统计接口：计算收支总额、分类占比、每日收支，提供图表数据源

### 4. AI智能分析模块（LangChain4j + 本地DeepSeek + Qdrant）
1. 接收客户端账单数据，构造Markdown结构化账单Prompt送入大模型
2. 输出标准化财务诊断、省钱建议、月度复盘文本
3. Qdrant向量库存放用户长期消费习惯向量，实现个性化持续优化分析
4. 文本分类接口：解析账单备注，自动匹配最优消费分类

### 5. 文件与图片存储模块
1. 接收手机上传的小票图片，图片文件保存至电脑本地磁盘，MySQL仅存储文件访问路径
2. 账单Excel导出接口：后端动态生成Excel文件，提供文件下载接口

### 6. 安全与通用工具模块
1. 全局请求参数校验、统一异常捕获、SLF4J日志记录
2. CORS跨域配置，同时允许鸿蒙客户端、Web管理后台跨域访问
3. Knife4j自动生成接口文档，区分普通用户接口、管理员接口
4. 全局操作日志：记录管理员后台所有修改、删除操作

## 四、MySQL数据库6张核心数据表（交由AI完整生成建表语句）
1. `sys_user` 普通用户表
  字段：主键id、用户名、加密密码、创建时间、账号状态
2. `sys_admin` 管理员表
  字段：主键id、管理员账号、加密密码、角色权限、创建时间
3. `bill` 账单表
  字段：主键id、用户id、收支金额、收支类型(收入/支出)、分类id、文字备注、小票图片路径、消费时间、创建时间、更新时间
4. `category` 消费分类表
  字段：主键id、用户id、分类名称、图标标识、分类类型(收入/支出)
5. `budget` 月度预算表
  字段：主键id、用户id、统计年月、总预算金额、各分类预算JSON存储、创建时间
6. `save_target` 存钱目标表
  字段：主键id、用户id、目标名称、目标总金额、当前已存金额、完成状态、创建时间、完成时间

## 五、局域网联调部署规范
1. 电脑与手机/浏览器连接同一WiFi，后端固定端口8080，使用电脑WLAN IPv4作为服务访问地址
2. Windows防火墙放行8080端口，关闭第三方杀毒软件端口拦截
3. 离线逻辑：无网络时账单存入手机Preferences离线队列；恢复网络自动批量调用后端同步接口写入MySQL
4. 全部业务数据唯一持久化位置为电脑MySQL，手机本地仅临时缓存离线未同步数据，无独立本地数据库存储账单
5. 管理员后台访问地址：`http://电脑内网IP:8080/admin`，浏览器直接打开使用
